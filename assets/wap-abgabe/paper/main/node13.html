Embarrisingly-Parallel

Zum Glück sind eine ganze Menge an Problemen ganz einfach parallelisierbar. Hierbei lassen sich zum Beispiel die Eingangsdaten einfach partitionieren, wobei dann für jede Partition ein Ergebniss unabhängig von den anderen Partiionen errechnet werden kann. Beispiele hierfür wären zum Beispiel das Brute-force raten des Eingangswertes einer Einwegfunktion (Passwort cracking). Auch im 3D Rendering bei der Verwendung von Ray Tracing ist es ganz einfach nachvollziehbar, dass das Ergebniss mehrere Lichtstrahlen in einer gegebenen Umgebung parallel berechnet werden kann.
